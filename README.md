# CS-360 Mobile App Reflection

## App Requirements and Goals

The goal of this project was to design and develop a functional mobile inventory management application for Android. The application was intended to help users track inventory items, monitor stock quantities, and receive notifications when inventory levels became low. This app was designed to address the user need for a simple and efficient way to manage item quantities without relying on manual tracking methods such as spreadsheets or handwritten records. By allowing users to store, update, and monitor inventory data directly from a mobile device, the application improves convenience and accessibility.

## User Interface Design and User Needs

Several screens and features were necessary to support user needs and create a user-centered design. These included a login and account creation screen, a dashboard displaying inventory items, an add/edit item screen, and an SMS notification screen. The login system provided account security and personalized access to stored inventory data. The dashboard served as the central location for viewing and managing items, while the add/edit screen allowed users to easily update inventory records. SMS notifications helped users stay aware of low-stock items.

The UI design focused on simplicity, clear navigation, and ease of use. Screen layouts were organized to ensure important actions such as adding items, editing records, and checking notifications were easy to find. Spacing, button placement, and visual consistency were carefully considered to reduce confusion and improve usability. These design choices were successful because they allowed users to navigate the application with minimal friction.

## Development Approach

I approached the coding process by breaking the application into smaller functional components and implementing features incrementally. Rather than building every feature at once, I focused on core functionality first, such as database setup, user authentication, and inventory management, before moving on to additional features like SMS notifications and UI refinements.

A key strategy I used was continuous testing and iterative improvement. After implementing each feature, I tested it to verify expected behavior before moving forward. This helped reduce larger debugging issues later in development. This structured approach can be applied to future software projects because it improves organization, simplifies troubleshooting, and makes complex projects more manageable.

## Testing and Validation

Testing played an essential role in ensuring the application functioned correctly. I tested each screen and feature individually using the Android emulator, verifying navigation, account creation, data persistence, item editing, item deletion, and low-inventory alerts. I also used debugging tools to identify and resolve runtime issues during development.

This process was important because even small coding errors can cause unexpected application behavior or crashes. Testing revealed issues related to navigation flow, UI behavior, and data persistence that required adjustments. By identifying these issues early, I was able to improve both functionality and user experience.

## Challenges and Innovation

One challenge during development was ensuring smooth navigation and consistent behavior across multiple activities while maintaining persistent user and inventory data. Managing interactions between screens and ensuring data updates reflected correctly throughout the application required careful planning and debugging.

To overcome these challenges, I refined activity communication, improved database handling, and adjusted UI behavior to create a more seamless experience. This process required problem-solving and adaptability, especially when unexpected bugs appeared during testing.

## Demonstrated Skills and Success

The component where I was most successful in demonstrating my knowledge and technical skills was the integration of database-driven inventory management with user interaction. Successfully implementing features such as account authentication, persistent inventory storage, item editing, deletion, and notification logic demonstrated my understanding of mobile application architecture and software design principles.

This project strengthened my skills in application design, debugging, testing, and user-centered development. It also provided valuable experience in completing a full mobile app development cycle, from planning and UI design to final implementation and refinement.
